# Flujo de trabajo: Figma a Codex/Claude Code vía Magic Path

## Resumen general
El vídeo muestra un flujo de trabajo gratuito para transformar diseños de Figma en código funcional ejecutable en local mediante **Codex** o **Claude Code**, utilizando **Magic Path** como intermediario. El proceso comienza instalando una *skill* (habilidad) en el agente de IA, autenticando Figma y pegando el enlace del frame seleccionado. Magic Path importa el diseño, permite generar variantes de estilo y crear nuevas pantallas contextuales (como una página de blog o dashboard) con mayor riqueza visual que si se pidiera directamente al agente de código. Finalmente, el componente seleccionado en Magic Path se envía a Codex/Claude Code para generar la implementación real (React, iOS, Electron) y previsualizarla en *localhost* o simulador.

## Ideas principales
- **Puente Figma → Código:** Magic Path actúa como capa intermedia que interpreta el diseño y genera contexto semántico (no solo píxeles) para los agentes de codificación.
- **Instalación única:** Se instala una *skill* externa copiando un comando desde la documentación de Magic Path (sección *External Agents*) y reiniciando el agente (Codex o Claude Code).
- **Autenticación Figma:** Requiere autorización única vía OAuth al pegar el primer enlace de selección (`Copy link to selection` / `Cmd+L`).
- **Generación de variantes:** Magic Path permite crear variantes de estilo (colores, disposición, *neo-brutalism*, etc.) a partir de un mismo frame.
- **Contexto superior:** Al pedir nuevas pantallas (ej. página de blog, dashboard) dentro de Magic Path, este infiere estructura de contenido (listas, ingredientes, artículos relacionados) que el agente de código solo no resuelve bien con *prompts* simples.
- **Ciclo iterativo:** Seleccionas componente en Magic Path → Pides a Codex/Claude Code que lo implemente/reemplace → Se ejecuta en *localhost* / Xcode / Simulador.
- **Multiplataforma:** El mismo flujo sirve para web (React/Next.js), apps móviles (iOS/SwiftUI via Xcode) y apps de escritorio (Electron).
- **Limitación cuenta gratuita:** El nivel *free* permite probar el flujo con unos pocos proyectos/sitios, no uso ilimitado.

## Conceptos, nombres, herramientas, empresas, fechas y cifras relevantes
- **Herramientas principales:** Figma, Magic Path (magicpath.ai), Codex, Claude Code, Xcode.
- **Tecnologías objetivo:** React / Next.js (web), SwiftUI (iOS), Electron (desktop).
- **Comandos clave:** `Authenticate Me`, `install this skill`, `can you see the selected component?`, `build this design and run it on the local host`, `replace the blog template page with the design of the selected Magic Path component`.
- **Atajo Figma:** `Cmd + L` (Copy link to selection).
- **Comunidad del autor:** Discord "Creator Network" (enlace en descripción del vídeo).
- **Estilos de ejemplo generados:** *Neo-brutalism*, variantes de tarjetas con etiquetas repositionadas.
- **Coste:** Flujo gratuito con límites de cantidad de proyectos en plan *free*.

## Conclusión breve
Magic Path cubre la brecha semántica entre el diseño visual (Figma) y la generación de código (Codex/Claude Code), permitiendo iterar diseños, crear variantes y generar pantallas contextuales completas antes de pasar a la implementación real. El flujo es gratuito para empezar, multiplataforma y reduce drásticamente la complejidad de los *prompts* necesarios al agente de código.
