# Documento Deprecado

Este documento queda sustituido por:

- `docs/planificacion-maestra-v2.md`

Motivo:

- la propuesta `docs/cambios-propuestos-v2.md` cambia la estrategia del
  proyecto hacia una arquitectura local-first con API fallback;
- el roadmap anterior ya no refleja con suficiente precision el objetivo real
  de dependencias, providers, cache, coste, privacidad y escalabilidad;
- mantener dos planificaciones activas generaria ambiguedad tecnica.

Uso recomendado:

- consultar `docs/planificacion-maestra-v2.md` como documento rector;
- consultar `docs/specs/current-pipeline.md` como contrato del comportamiento
  actualmente implementado;
- consultar `docs/cambios-propuestos-v2.md` como fuente de la propuesta de
  cambio estrategico.
